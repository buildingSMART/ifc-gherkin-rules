GEM011 Curve segments consistency - v1

001 | @implementer-agreement
002 | @GEM
003 | @version1
004 | @E00010
005 |
006 | Feature: GEM011 - Curve segments consistency
007 |
008 |   The rule verifies that the same entity type is used for all segments
009 |   and that IfcCurveSegment is used with IfcSegmentedReferenceCurve and IfcGradientCurve
010 |   The rule is identical to ALS011, except that it does not require the presence of an IfcAlignment.
011 |
012 |
013 | Background:
014 |   Given a model with Schema 'IFC4.3'
015 |   Given an .IfcProduct.
016 |   Given [its entity type] ^is not^ .IfcAlignment.
017 |   Given its attribute .Representation.
018 |   Given its attribute .Representation.
019 |   Given its attribute .Items.
020 |
021 |
022 | Scenario Outline: Consistent entity types used
023 |
024 |   Given an .<entity>.
025 |   Given its attribute .Segments.
026 |   Given its entity type
027 |
028 |   Then The values must be identical at depth 1
029 |
030 |   Examples:
031 |     | entity            |
032 |     | IfcCompositeCurve |
033 |     | IfcGradientCurve  |
034 |     | IfcSegmentedReferenceCurve |
035 |
036 |
037 | Scenario Outline: IfcCurveSegment used for IfcSegmentedReferenceCurve and IfcGradientCurve
038 |
039 |   Given an .<entity>.
040 |   Given its attribute .Segments.
041 |   Given its entity type
042 |
043 |   Then The value must be 'IfcCurveSegment'
044 |
045 |   Examples:
046 |     | entity            |
047 |     | IfcGradientCurve  |
048 |     | IfcSegmentedReferenceCurve |
