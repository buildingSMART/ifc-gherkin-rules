   005 | Feature: BLT001 - Correct use of operation type attributes for doors
   006 | 
   007 |   The rule verifies that attribute UserDefinedOperationType is provided only when
   008 |   the value of the attribute OperationType is set to USERDEFINED.
   009 |   Additionally, when an IfcDoor is assigned an IfcDoorType, the type should
   010 |   dictate the OperationType, and the instance should not override it.
   011 | 
   012 | Scenario Outline: Correct values for OperationType and UserDefinedOperationType
   013 |
