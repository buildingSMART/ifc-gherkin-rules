   006 | Feature: BLT001 - Correct use of operation type attributes for doors
   007 | 
   008 |   The rule verifies that attribute UserDefinedOperationType is provided only when
   009 |   the value of the attribute OperationType is set to USERDEFINED.
   010 |   Additionally, when an IfcDoor is assigned an IfcDoorType, the type should
   011 |   dictate the OperationType, and the instance should not override it.
   012 | 
   013 | Scenario Outline: Correct values for OperationType and UserDefinedOperationType
   014 |
