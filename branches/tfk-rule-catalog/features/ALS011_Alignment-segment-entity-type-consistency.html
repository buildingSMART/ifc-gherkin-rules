005 | Feature: ALS011 - Alignment segment entity type consistency
006 |
007 |   The rule verifies that the same entity type is used for all segments
008 |   and that IfcCurveSegment is used with IfcSegmentedReferenceCurve and IfcGradientCurve
009 |   The rule is similar to GEM011, except that it requires the presence of an IfcAlignment.
010 |
011 |
012 | Background:
013 |   Given a model with Schema 'IFC4.3'
014 |   Given an .IfcAlignment.
015 |   Given its attribute .Representation.
016 |   Given its attribute .Representation.
017 |   Given its attribute .Items.
018 |
019 |
020 | Scenario Outline: Consistent entity types used - direct representation
021 |
022 |   Given an .<entity>.
023 |   Given its attribute .Segments.
024 |   Given its entity type
025 |
026 |   Then The values must be identical at depth 1
027 |
028 |   Examples:
029 |     | entity            |
030 |     | IfcCompositeCurve |
031 |     | IfcGradientCurve  |
032 |     | IfcSegmentedReferenceCurve |
033 |
034 |
035 |   Scenario Outline: Consistent entity types used - capture parent curve in case it is not a direct representation
036 |
037 |     Given an .<entity>.
038 |     Given its attribute .BaseCurve.
039 |     Given an .<entity_parent>.
040 |     Given its attribute .Segments.
041 |     Given its entity type
042 |
043 |     Then The values must be identical at depth 1
044 |
045 |     Examples:
046 |       | entity                      |  entity_parent |
047 |       | IfcGradientCurve            |  IfcCompositeCurve |
048 |       | IfcSegmentedReferenceCurve  |  IfcCompositeCurve |
049 |
050 |
051 | Scenario Outline: IfcCurveSegment used for IfcSegmentedReferenceCurve and IfcGradientCurve
052 |
053 |   Given an .<entity>.
054 |   Given its attribute .Segments.
055 |   Given its entity type
056 |
057 |   Then The value must be 'IfcCurveSegment'
058 |
059 |   Examples:
060 |     | entity            |
061 |     | IfcGradientCurve  |
062 |     | IfcSegmentedReferenceCurve |
