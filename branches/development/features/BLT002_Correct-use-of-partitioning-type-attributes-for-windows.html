005 | Feature: BLT002 - Correct use of partitioning type attributes for windows
006 |
007 |   The rule verifies that attribute UserDefinedPartitioningType is provided only when
008 |   the value of the attribute PartitioningType is set to USERDEFINED.
009 |   Additionally, when an IfcWindow is assigned an IfcWindowType, the type should
010 |   dictate the PartitioningType, and the instance should not override it.
011 |
012 | Scenario Outline: Correct values for PartitioningType and UserDefinedPartitioningType
013 |
014 |   Given an .<entity>.
015 |   Given .UserDefinedPartitioningType. ^is not^ empty
016 |   Given .IsTypedBy. ^is^ empty
017 |
018 |   Then .PartitioningType. ^is^ 'USERDEFINED'
019 |
020 |   Examples:
021 |     | entity |
022 |     | IfcWindow |
023 |     | IfcWindowType |
024 |
025 |
026 | Scenario: Correct IfcPartitioningType with relating type object
027 |
028 |   Given an .IfcWindow.
029 |   Given a relationship .IfcRelDefinesByType. to .IfcWindow. from .IfcWindowType.
030 |
031 |   Then .PartitioningType. ^is^ empty
